# 仕様書：STRETCH（仮題）

> スマホで遊べるWebアプリ（HTML5）。
> コンセプト：**簡単操作で短時間、ほどよいランダム性と中毒性で何度も遊びたくなる**。
> 設計の全判断は [面白いゲームを作るための指針.md](面白いゲームを作るための指針.md) に紐づく（各項に「→指針N」で対応を明記）。

---

## 0. ひとことコンセプト（指針10：コンセプトの一貫性）

> **「電車の中で、片手・親指1本、30秒〜2分。棒をギリギリまで伸ばして次の足場へ渡る。欲張るほど高得点、外せば即終了。あと1回がやめられない。」**

- 誰が：スキマ時間のスマホユーザー（年齢問わず）
- どこで：移動中・待ち時間
- どう遊ぶ：画面を押して棒を伸ばし、指を離して倒す。届けば前進、外せば落下で終了
- どう楽しむ：**ギリギリを攻める快感**（リスク&リターン）と**ハイスコア更新**

このコンセプトは開発中に変えない（指針10）。

---

## 1. ゲーム性の核：リスクとリターン（指針1）

**かけひき = 「棒をどこまで伸ばすか」の一点に集約する。**

| 要素 | 内容 |
|------|------|
| リスク | 短すぎると届かず落下／伸ばしすぎても足場を越えて落下＝**即ゲームオーバー** |
| リターン | 足場に届けば前進＝スコア。さらに**足場中央の「パーフェクトゾーン」**に倒すとボーナス＆コンボ倍率上昇 |
| プレイヤーの裁量 | 「安全に足場の手前側を狙う」か「中央パーフェクトを攻める」かを毎回選べる（指針1：どこまでリスクを取るか） |
| 神プレイ | パーフェクト連続でコンボ倍率が伸び続ける＝最高リスク×最高リターンの瞬間（指針1） |

---

## 2. ゲームループ（指針2：ストレス→解消→成長→進展）

```
次の足場が見える（ストレス：距離が読めない）
  → 押して棒を伸ばし、離す（解消：自分の判断で渡る）
  → 渡れた＆パーフェクト（成長：コンボ倍率が上がる）
  → スコア加算・前進・足場が遠ざかる（進展：難度がじわ上がる）
  → また次の足場（新たなストレス）
```

- 1アクション（押す→離す）で1サイクルが完結。テンポ最優先（指針8）。
- ゲームオーバー後は**0.5秒以内に即リトライ可能**にして「もう1回」を阻害しない（指針2：やり直しの動機付け）。
- 失敗時は「あと少しで届いた」が一目で分かる（棒が倒れて足場に少し届かない絵）＝「次はできそう」を演出（指針2）。

---

## 3. 主軸の面白さ（指針3：7種類のうち①「操作自体が楽しい」）

- 主軸は**①操作自体が楽しい**（タイミングアクション）に全振り。
- ストーリー・キャラ・音ゲー要素は持ち込まない（MVPでは）。迷ったらこの軸に戻る。

---

## 4. 最初の3分（指針4）

1. 起動 → **タップ1回でゲーム開始**。スプラッシュは最小（ロゴ0.5秒以内、タップでスキップ）。
2. オープニング映像・別画面チュートリアルは**作らない**。
3. **チュートリアル＝本編**：最初の1〜2足場だけ「長押し」「離す」の半透明ガイドを重ねる。1回成功したら自動的に消える（指針4：練習している感をなくす）。
4. 冒頭からゲームの核（棒を伸ばして渡る快感）をいきなり体験させる。

---

## 5. 報酬設計（指針5）※MVPはスコア特化

| 報酬 | 内容 |
|------|------|
| ハイスコア | localStorage に保存。タイトル＆結果画面に常時表示（次の目標が常に見える＝指針5「目の前に吊られたごほうび」） |
| コンボ倍率 | パーフェクトを連続するほど ×1 → ×2 → ×3… と上昇。失敗（非パーフェクトで着地）でリセット |
| その場の達成 | 1足場ごとに必ずスコアが入る＝セッション毎に必ず何か得られる（指針13） |
| 自己ベスト演出 | スコアが自己ベストを超えた瞬間に専用エフェクト＆SE（指針12：プレイヤーを褒めるUI） |

**将来拡張（今は作らない／後付けで崩れない設計にしておく）**：コイン→スキン/テーマのアンロック（コレクション）、デイリーチャレンジ。スコア計算とアンロックを疎結合にしておく。

---

## 6. フィードバック＝手ごたえ（指針6）

入力に対して**わかりやすく・おおげさに**返す（指針6：情報が減るからおおげさに）。

| タイミング | 演出 |
|-----------|------|
| 押している間 | 棒がにょきにょき伸びる＋「伸びる音」をピッチ上昇で鳴らす（チャージ感）＋わずかな緊張BGM/ティック |
| 離した瞬間 | 棒が「パタン」と倒れるモーション＋効果音 |
| 着地成功 | 着地点に小エフェクト＋「コツッ」音＋キャラが歩いて渡る |
| **パーフェクト** | **ヒットストップ（数フレーム停止）**＋画面フラッシュ＋「キラッ」音＋コンボ数ポップ＋端末振動（`navigator.vibrate`）（指針6） |
| 落下（失敗） | **画面シェイク**＋落下音＋キャラが落ちる＋軽い振動（指針6） |
| 自己ベスト更新 | 紙吹雪エフェクト＋ファンファーレ（指針5・12） |

---

## 7. サウンド（指針7）

- **音はフィクション**＝現実音でなく「気持ちいい音」優先（指針7）。
- 実装は **Web Audio API でシンセ生成**（外部音源ファイル不要・ロード待ちゼロ＝指針8）。
  - チャージ音：押している間ピッチが上がるサイン波
  - 倒す/着地/パーフェクト/落下：短い矩形波・ノイズで作る
- BGMはMVPでは省略可（後付けでループ設計）。重要な音（パーフェクト）が埋もれないよう音量管理（指針7）。
- 端末ミュート/無音スイッチ対応。設定で音ON/OFFトグル。

---

## 8. テンポ（指針8：遅さは罪）

- **60fps目標**（`requestAnimationFrame`）。
- 画面遷移・UIアニメは **0.2秒以内**（指針13）。
- ロード待ち体感ゼロ：単一HTML＋Canvas、外部アセット最小、初回描画まで一瞬。
- ゲームオーバー→リトライは**待たせない**（タップで即再開）。

---

## 9. 難易度・ランダム性（指針9）

- **ランダムは「結果」でなく「場面の多様性」に使う**（指針9）。乱数で勝手に落ちることはない＝公平。
  - ランダム対象：**次の足場の幅**と**足場までの距離（=必要な棒の長さ）**。
- **段階的に難化**：進むほど (a) 足場が細くなる (b) 足場間距離のばらつきが広がる。ただし下限・上限を設けて理不尽にしない。
- **ズルの余地**（指針9）：足場の「手前ギリギリ端」を狙う安全ルートも成立させる（パーフェクトは取れないがコンボを繋がず生存重視）。上級者は中央パーフェクト連打で青天井スコア＝「それぞれの勝ち方」（指針9）。

### 数値パラメータ（初期案・調整前提）

| パラメータ | 初期値 | 備考 |
|-----------|--------|------|
| 棒の伸びる速度 | 約 300 px/秒 | 押し続けで線形に伸長 |
| 足場の幅 | 開始 60px → 最小 24px | スコアに応じて漸減 |
| 足場間ギャップ | 50〜180px（序盤）→ 最大320px | 進むほど上限拡大 |
| パーフェクトゾーン幅 | 足場中央 ±8px | 一定（細い足場ほど相対的に難） |
| コンボ倍率 | パーフェクト毎 +1（上限なし） | 非パーフェクト着地でリセット |
| スコア | 着地=+1、パーフェクト=+ (1 × コンボ倍率) | 例：3連パーフェクトで +3 |

> これらは「触って気持ちいい」を最優先に**実機で調整**する前提のたたき台。

## 10. 操作仕様（指針12：判定を明確に）

- **押す（pointerdown / touchstart）**：棒が伸び始める。
- **離す（pointerup / touchend）**：棒が倒れる。倒れた長さで判定。
- 片手・親指1本で完結。画面どこを押してもOK（押す位置は判定に無関係）。
- マルチタッチは最初の1点のみ採用。
- PCでもマウス押下/離しで同じ操作（開発・テスト用）。

### 判定ロジック
1. 棒の長さ L、自キャラから次足場の手前端まで D_near、奥端まで D_far。
2. `D_near ≤ L ≤ D_far` → 着地成功。
3. 着地点が `中央 ± パーフェクト幅` → パーフェクト（ボーナス＋コンボ）。
4. それ以外 → 落下＝ゲームオーバー。

## 11. 画面・UI（指針12）

縦持ち（ポートレート）固定。横画面は「縦にしてね」案内 or レターボックス。

```
┌──────────────┐
│  SCORE 12      BEST 87 │  ← 常時表示・大きめ文字（指針12：視認性最優先）
│                        │
│        COMBO ×3        │  ← パーフェクト中だけ大きく出る
│                        │
│                        │
│   ｵ                    │  ← キャラ
│  ■■■───┐      ■■■      │  ← 現在の足場 / 伸びる棒 / 次の足場
│  └────────────────┘    │
└──────────────┘
```

- 画面構成は3つだけ：**タイトル / プレイ中 / 結果**。それぞれ0.2秒以内で遷移。
- タイトル：ゲーム名 + 「タップでスタート」+ BEST表示。
- 結果：今回スコア・BEST・（更新時は祝福演出）・「もう一度」ボタン（実際はタップ即再開）。
- 文字は大きく、当たり判定（パーフェクトゾーン）は視覚的に明示（足場中央に細いマーカー）。

## 12. 技術仕様

| 項目 | 決定 |
|------|------|
| 形態 | 単一の静的Webアプリ（ビルド不要）。`index.html` に CSS/JS インライン or 数ファイル |
| 描画 | HTML5 **Canvas 2D**（`requestAnimationFrame`、60fps） |
| 言語 | Vanilla JavaScript（フレームワーク不要・ロード軽量＝指針8） |
| 音 | Web Audio API（シンセ生成、外部ファイルなし） |
| 保存 | localStorage（ハイスコア・設定） |
| 入力 | Pointer Events（touch/mouse 統一） |
| 振動 | `navigator.vibrate`（対応端末のみ、未対応は無視） |
| 解像度対応 | `devicePixelRatio` 対応でRetinaくっきりＡ。リサイズ追従 |
| 対象 | モバイルSafari / Chrome を主、PCブラウザも動作 |
| オフライン | 後で Service Worker / PWA 化（manifest + アイコンで「ホーム追加」）※MVP後 |
| デプロイ | 静的ホスティング（GitHub Pages / Netlify / Cloudflare Pages いずれも可） |

### ファイル構成（案）
```
game/
  index.html      … 画面・canvas・UI
  style.css       … レイアウト（縦持ち・セーフエリア対応）
  game.js         … ゲームループ・状態管理・判定・描画
  audio.js        … Web Audio のSFX生成
  仕様書.md
  面白いゲームを作るための指針.md
```

## 14. 「面白さ」チェックリスト（リリース判定用・指針より）

- [ ] タップ1回で3秒以内に遊び始められるか（指針4）
- [ ] 「どこまで伸ばすか」の判断に毎回ドキドキがあるか（指針1）
- [ ] パーフェクト時に「気持ちいい！」と声が出る手ごたえか（指針6・7）
- [ ] ゲームオーバー後すぐ「もう1回」を押したくなるか（指針2）
- [ ] 待ち時間・カクつきがないか（60fps / 0.2秒遷移）（指針8）
- [ ] ランダムで理不尽に死なないか（公平性）（指針9）
- [ ] ハイスコアという次の目標が常に見えているか（指針5）
